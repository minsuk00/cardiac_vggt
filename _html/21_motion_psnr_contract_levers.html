Question

A chain of prior investigations proved this cardiac slice-to-volume model (VGGT + 3-D refiner) is warp-limited: a perfectly-optimized deformable warp of the ~10 sparse single-frame inputs tops out at ~21 dB motion PSNR (≈ the trained model), ~14 dB below a perfect-placement oracle (35). The 14 dB is the appearance wall — warping other-phase intensities can't synthesize target-phase appearance — and the dominant ~7-9 dB of it is subject-specific appearance simply not present in the sparse inputs. Two proposed fixes were then tested and refuted: appearance synthesis (feature-splat + decoder, +0.03 dB) and native-resolution rendering (−0.13 dB). The diagnosis said the real lever is the input contract — but that was reasoned, never measured.

So: what input-contract change actually raises MOTION PSNR, and by how much? Test it, give a conclusive, actionable answer. (No retraining — probe the existing model by rebuilding input compositions.)

Conclusion (TL;DR)

There is no large lever within the fast-acquisition contract. Motion PSNR is bound by how much of the target cardiac phase you DIRECTLY observe, and the model does not propagate that appearance to planes it didn't measure. Two training-free experiments (24 + 8 held-out subjects, prove-it-clean) settle it: (1) Adding input frames at the target phase raises full-volume motion PSNR (+2 dB) — but this is a pure observation leak: the leak-free held-out-z motion PSNR (planes given no target frame) falls (18.2→13.6). The model only fills the planes you directly measure; it cannot synthesize the target phase on unobserved planes. (2) The one cheap, goal-compatible lever is proximity sampling — biasing the scattered acquisition toward the target phase band (companions at t±1 instead of random) buys +0.8 dB (18.2→19.0), and the warp ceiling gain is +0.84 dB, so the current model already extracts essentially all of it — no retrain headroom. Everything else (renderer, decoder, splat-res, input-t) is ≤0.1 dB. Actionable answer: the only free win is proximity sampling (~+0.8 dB); a large motion-PSNR gain requires breaking the contract — directly measuring more of the target phase (more frames/slice near target, a temporal stream, or a 5-D reference) — and even then it helps only the planes you actually measure.

Confidence: high. The headline rests on a leak-free held-out-z metric (verified to have ~1e-16 splat z-bleed, so it measures genuine propagation), a faithful-to-6e-8 reconstruction of the real input pipeline, and a warp-ceiling A/B whose +0.84 dB is robust to 2.7× more optimization (+0.016 dB). All three load-bearing scripts passed an adversarial prove-it with clean bills. Main bound: in-distribution simulated data, clean (no-respiratory) protocol, S=8.

The toy

The model's input is decoupled (the target phase is a query; input slices are sampled at random phases & z). That lets me probe the contract without retraining: rebuild input compositions straight from each subject's cached phase bundle and read motion PSNR. Two levers, both clean protocol, fixed budget S=8, fixed per-subject z-set, paired across conditions:

  • Lever A — COUNT at target. Place K of the 8 input slices exactly at the target phase (distinct z),

the rest at random other phases. Sweep K=0,1,2,4,8. The trap (flagged by the red-team): scoring motion PSNR on a plane where you injected a true target-phase slice is the oracle by construction. So I report both the leaky full-volume number and the leak-free held-out-z number (dynamic voxels only on planes that got no target slice) = genuine propagation skill.

  • Lever B — PROXIMITY at fixed budget. K=0 (no exact target frame), identical z-set, only the companion

phases change: random vs t±2 vs t±1. Leak-free by construction. Tests whether which phases you sample beats how many frames.

  • Follow-up — warp ceiling by composition. Direct-optimize a free per-pixel Δ (the best-possible warp)

on random vs near±1 inputs, to ask whether proximity's small model gain is a floor (higher ceiling → worth a retrain) or capped at the wall (+0.8 is the whole prize).

Hypotheses & debate

Four agents argued it out and converged on the discriminating tests:

  • H1 (more / target-phase frames): the only lever is observing more of the target phase; predicted a

steep concave rise (K=1≈21 → K=4≈28-30 → K=all≈35). Its hidden assumption: that the information ceiling equals the realizable gain — the model (trained on ~1 target frame) might ignore extra ones.

  • H3 (proximity): at fixed budget, near-phase companions beat random because appearance error grows with

|Δt|; pointed at subject_temporal_interp=28.1 (from dense t±1) and predicted ≥+5 dB for near±1 vs random.

  • H2 (input-t knowledge): giving the model the input phase lets it interpret frames better — but a prior

eval found input-t neutral/harmful; the panel judged it weak alone (≤+0.5 dB).

  • Red-team: the K-sweep is an oracle leak (you score where you inject) and confounds z-coverage —

mandated the held-out-z metric, fixed z-set, per-composition identity floors, and a ceiling check to tell "retrain" from "give up."

The experiments came in below both H1's and H3's predictions: the K-gain is leak (held-out falls), and proximity is real but +0.8, not +5.

Experiments & results

Lever A — target-phase frames only fill what you measure (no propagation)

Adding input frames at the target phase raises FULL-volume motion PSNR (18.2→20.1) but that is anhigh

observation leak; on planes the model didn't directly measure, it gets WORSE (18.2→13.6). The full-volume curve (blue) rises +2 dB with K — but the leak-free held-out-z curve (red), which scores only dynamic voxels on z-planes that received no target-phase slice, declines monotonically to 13.6. So the +2 dB is entirely "scoring the answer where you injected it." The model does not propagate target-phase appearance to neighboring planes — exactly the appearance wall, now shown from the acquisition side. (The held-out-z metric is genuinely leak-free: identity-splat z-bleed is ~1e-16, verified in prove-it.)

Lever B + ceiling — proximity is the only free lever, and it's small & capped

Sampling companions NEAR the target phase (t±1) beats random by +0.8 dB at equal frame count — and the modelhigh

already extracts essentially all of it. Leak-free (K=0, identical z-set), the trained model goes random 18.2 → near±2 18.6 → near±1 19.0 (+0.8 dB). The best-possible WARP under the same compositions goes random 19.1 → near±1 19.9 (+0.84 dB ceiling gain). Since the model's realized gain (+0.8) ≈ the ceiling gain (+0.84), the current model is already at its proximity ceiling — a retrain would buy ~nothing. Proximity is real, cheap, and compatible with the fast-acquisition goal (no extra frames, just which phases you sample) — but it's a ~+0.8 dB lever, not a breakthrough, and it does not reach even the prior ~21 dB warp ceiling.

Where every lever lands

Every model-, renderer-, and sampling-side lever lands within ~1 dB of the wall; the only large headroom ishigh

direct observation of the target phase (28.1 dense-temporal, 35 oracle). Renderer/decoder (+0.03), splat-resolution (−0.13), input-t (~0), proximity (+0.8) — all clustered just above the ~18-21 dB contract floor. The big numbers (28.1 from the subject's own DENSE t±1 volumes; 35 perfect placement) require information the sparse, blind, ~1-frame-per-slice contract does not contain.

Confirmation at scale

The two experiments cross-validate (Lever B's +0.80 model gain ≈ the independent +0.84 warp-ceiling gain), and the no-propagation result is consistent with every prior finding (warp ceiling ~21, appearance synthesis +0.03, population template 14.4). The conclusion is an information statement, not a model that more training/scale would lift: the missing target-phase appearance on unobserved planes is not in the inputs, so no amount of training conjures it (prior work already showed synthesis hallucinates it, PSNR-neutral). What remains unverified: real (non-simulated) data, the breathing protocol (would shift absolute numbers down but not the ordering), and whether a deliberately propagation-trained model could beat its warp ceiling on held-out planes (bounded above by the same wall, so expected small).

Conclusion & confidence

What you need to do for better motion PSNR, ranked by evidence:

  1. Proximity sampling — the only free win (~+0.8 dB). Bias the scattered single-frame acquisition (or the

query) toward the target cardiac-phase band (±1 phase). No extra frames, no retrain (the model is already at this ceiling). Cheap and goal-compatible, but modest.

  1. For a LARGE gain, you must break the fast-acquisition contract — directly observe more of the target

phase: more frames per slice clustered near target, a short temporal stream per slice, or a 5-D motion-resolved reference. This is the only path to the 28-35 dB regime, and it trades away acquisition speed. Even then it helps only the planes you measure — the model won't propagate to unobserved planes.

  1. Stop tuning the model/renderer for motion PSNR. Renderer, decoder, splat-resolution, input-t

conditioning are all ≤0.1 dB — proven dead here and in docs 19/20.

The honest headline: the ~21 dB wall is an acquisition-coverage wall, not a model wall. Within "one fast frame per slice, phase-blind," motion PSNR is essentially maxed; the lever is more/closer target-phase observation, which is a deliberate speed/quality trade, plus a small free proximity gain. (Residual: all in-distribution & clean-protocol; a propagation-trained model is unverified but ceiling-bounded.)
