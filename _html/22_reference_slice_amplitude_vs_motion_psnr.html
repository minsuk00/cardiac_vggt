Question

The user asked for a concrete, proven direction to improve motion correction — and flagged an apparent contradiction in prior results ("you said more frames won't help, but also that they do"), plus a specific idea: "if I give the model a slice with the LV state I want it to reconstruct, would that make motion PSNR much better?" Settle it by experiment, not opinion: does a target-phase reference slice improve reconstruction, and on which metric?

Conclusion (TL;DR)

Your instinct is right — but for the RIGHT metric. A target-phase reference slice ("the LV state you want") fixes the contraction AMPLITUDE the model currently gets badly wrong (per-patient EF), but it does NOT meaningfully improve per-voxel held-out MOTION PSNR. Two things are true and not contradictory: (1) The model regresses everyone's contraction amplitude to the cohort mean (~flat EF); a reference slice recovers per-patient amplitude (toy: pred-EF-vs-true slope 0.26 → 0.73; prior real-model coverage ablation → 1). (2) But the held-out per-voxel motion error is appearance PATTERN, not magnitude: M_pred vs M_gt per-plane cosine is only 0.44, so even a PERFECT amplitude correction (per-plane oracle) adds +0.25 dB, and the realistic one-reference-plane version hurts (−0.3 dB). 98% of the held-out motion error is not amplitude. So the proven direction is: condition on a target-phase reference slice (Option B) to fix per-patient contraction amplitude/EF — the clinically meaningful measure of motion-correction accuracy — and stop expecting per-voxel motion PSNR to move, because that is information-limited (the appearance wall), fixable only by directly observing more of the target phase.

Confidence: high for the scoped claims. The negative (amplitude won't move held-out motion PSNR) is a leak-controlled upper bound — even GT-fitted per-plane α adds ≤0.25 dB — and prove-it confirmed every risky design choice biases toward a larger apparent gain, so the small number is conservative. The positive (reference fixes EF) reproduces a prior real-model result. Scope: in-distribution, clean (no-resp), one checkpoint; the amplitude channel specifically (see caveats).

"More random frames" or appearance-synthesis = dead (docs 19/20). "More target-phase observation" recovers amplitude on the observed planes (so EF, a volume-integrated number, recovers) — but does not propagate appearance to unobserved planes (so held-out motion PSNR doesn't move). The toy / experiments

Three pieces, fastest first:

  • Contraction toy (toy_contraction.py, synthetic, known EF): compares conditioning schemes. C0 = current

(target_t index, blind phase); Ccov = index + 1 target slot; B = reference slice (frame-0), no index. Metric = pred-EF-vs-true-EF slope.

  • Amplitude-propagation test (toy_amplitude_propagation.py, real model, no retrain, leak-controlled): the

decisive one. Run the model twice on the SAME blind input (val RNG is seq-seeded ⇒ bit-identical inputs, runtime-verified): target=t → V_pred_t, target=ED → V_pred_0. M_pred = V_pred_t − V_pred_0 (model's predicted motion), M_gt = phases[t] − phases[0] (true motion). Ask: is the held-out error a propagatable global amplitude (one reference plane supplies a scalar α) or per-plane pattern? Score the motion residual α·M_pred vs M_gt directly (no volume reconstituted ⇒ no static-background leak), baseline α=1, on held-out planes (α fit on z_ref, scored on z≠z_ref). Controls: global-α oracle, per-plane-α oracle (regional ceiling), premise checks (per-plane cosine, CV of per-plane α).

  • prove-it on the load-bearing script (2 reviewers + per-phase audit).

Hypotheses & debate

  • H_ref (user's): the dominant recoverable error is a global contraction amplitude; one reference plane

supplies it → big held-out motion-PSNR gain.

  • H_local: the held-out error is per-plane appearance/unobserved info; no scalar fixes it → ~0 on held-out.
  • Red-team (pre-registered the design): the original plan had a fatal static-swap confound (anchoring

the corrected volume on GT ED while the baseline wasn't) — fixed by scoring the motion residual directly. It also demanded: prove the premise (cosine/CV) before the propagation test; test realistic z_ref (random, worst), not just best; stratify by phase; and recognize this answers the information question, not whether a retrain would learn it.

The premise check decided it before the propagation test even mattered.

Experiments & results

Two errors, two fates

A target-phase reference slice recovers per-patient contraction amplitude (EF slope 0.26→0.73) but adds athigh

most +0.25 dB to held-out motion PSNR — and the realistic one-plane version hurts. EF (left): current index scheme regresses to the cohort mean (slope 0.26); a reference slice (scheme B) recovers per-patient amplitude (0.73), consistent with the prior real-model coverage ablation (→1 when the target phase is observed). Held-out motion PSNR (right): baseline 20.42; a realistic 1-reference-plane α −0.28; the global-α oracle +0.08; even the per-plane-α oracle (each plane its own GT-fit scalar — an upper bound no reference could beat) only +0.25.

Why a scalar can't help: the held-out error is pattern, not magnitude

The model's predicted motion has the wrong spatial PATTERN (cosine 0.44), not just the wrong magnitude — so ahigh

reference's amplitude scalar cannot fix it, at any phase. Per-plane cosine(M_pred, M_gt) = 0.44 and CV of per-plane α = 0.52 ⇒ the premise of a single global amplitude fails. After the oracle global-α, 98% of the held-out error remains ⇒ amplitude is ~2% of it. Per-phase (right): at every target phase the 1-plane reference hurts and the oracle is flat; zero of 100 subject-phase samples gain >1 dB. No peak-systole win is averaged away.

Confirmation at scale & prove-it

The decisive test runs on the real trained model (no toy surrogate) over 20 held-out subjects × 5 phases; the linchpin (identical input across the two forwards ⇒ M_pred is pure model motion) was runtime-verified

(max|Δinput| = 0). prove-it (2 reviewers): no gain-suppressing bug; the two risky choices (model-ED anchor, GT-peeking oracles) both inflate the apparent gain, so ≤0.25 dB is a conservative upper bound; low cosine makes the null self-consistent.

Conclusion & confidence

The direction to take, proven:

  1. Condition on a target-phase reference slice (Option B), to fix per-patient contraction AMPLITUDE / EF.

This is the model's real, fixable error (it currently reports ~constant EF for everyone), and a reference recovers it (0.26→0.73; coverage→1). This is the clinically meaningful sense of "accurate motion correction."

  1. Do NOT expect per-voxel held-out MOTION PSNR to improve from this — its error is appearance pattern

(cosine 0.44), information-limited (the appearance wall), and the only lever is directly observing more of the target phase (the acquisition trade, doc 21). A reference's amplitude buys ≤0.25 dB even in the oracle.

  1. Reconsider the headline metric. Per-voxel motion PSNR conflates a fixable amplitude error with an

unfixable appearance-pattern error and is dominated by the latter; per-patient contraction amplitude (EF) is the quantity that is both wrong and fixable.

Scope & honesty (from prove-it): (1) The test isolates the amplitude/magnitude channel — it scores the motion residual with the static/ED reconstruction error differenced out, so it bounds the gain from rescaling predicted motion, NOT a reference improving static appearance at/near its own plane. (2) The appearance- propagation half of "would a reference help" is answered by doc 21 (target-phase frames don't propagate to held-out for this model); this experiment + doc 21 together cover the reference idea. (3) The per-plane-α oracle bounds only the magnitude-rescale family; a retrained reference-conditioned model could import pattern, but that is bounded by the recoverability ceiling (doc 20, ~28 dB with DENSE data, far less from one plane). (4) Cosine is computed against the model's own ED anchor (conflates pattern error with ED-recon error; the null is robust to this since a scalar fixes neither). (5) In-distribution, clean protocol, single noresp-aggft checkpoint.
