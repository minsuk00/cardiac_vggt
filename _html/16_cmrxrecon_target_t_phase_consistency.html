Is CMRxRecon's target_t = k/12 a fixed cardiac state?

Full-dataset analysis · 301 subjects · 12 cine phases each · gold standard = nnU-Net Task114 (M&Ms) LV blood-pool volume per phase. Generated by tools/cmrxrecon_phase_analysis/.

TL;DR & takeaway. No — a fixed interior target_t is NOT a fixed cardiac state across subjects. CMRxRecon's 12 phases are a resampling of each subject's own R–R interval (native phase count varies 14–40, never 12, yet all are stored as exactly 12), so target_t = k/12 is normalized fractional cycle time. Only target_t = 0 (end-diastole) is an anchored state (it is the LV-volume maximum for 97% of subjects). End-systole — measured by minimum LV blood-pool volume — drifts across frames 3–8 (fractional 0.25–0.67, IQR t5–t6, robust sub-frame std 0.68 frames). The ambiguity is bounded within this single breath-hold cohort (most subjects ES at t5–6) but real, and it is worst in mid-systole / mid-diastole. CMRxRecon has no ES label, so physiological ED↔ES phase normalization is impossible. Mixing a wider-heart-rate / cardiomyopathy cohort (e.g. ACDC) would widen this bounded ambiguity into a contradictory one.

Part 1 — The parameterization is normalized fractional time (exact, all 301)

Every subject has exactly 12 on-disk phase volumes (sax_frame_00..11), confirmed for 301/301. But the native acquisition (from each cine_sax_info.csv TemporalPhase field, the number of reconstructed cardiac phases at ~fixed ~50 ms temporal resolution per the CMRxRecon data paper) varies widely and is never 12:

A fixed output of 12 from a native count that is always >12 ⇒ the 12 phases are a temporal resampling of each subject's full R–R interval ⇒ phase k sits at fraction k/12 of that subject's own cycle.

Resampling over the full cycle — not truncation to the first 12

The mean image difference from end-diastole rises to a peak (≈ES) and then falls back toward ED for 100% of subjects — i.e. the 12 frames capture systole AND diastole (the whole cycle). First-12 truncation of a 14–40-frame acquisition would show only the rising limb. Corroborating: corr(intensity-ES-index, native phase count) = +0.31 (weak; truncation would force a strong positive correlation). The structural fact above is the airtight proof; this is corroboration.

Part 2 — Gold-standard cardiac state per phase (nnU-Net LV volume)

We segmented the LV blood pool in all 301×12 = 3612 phase volumes with the M&Ms challenge nnU-Net (Task114, 5-fold 2D ensemble; validated LV Dice 0.95). LV blood-pool volume is a direct, physiological cardiac-state measure. 301 subjects passed QC (0 skipped for empty/constant LV). Sanity: ejection fraction (argmax-ED) mean 64% ± 6% (gating-ED 64% ± 6%), physiologically normal; the LV-volume maximum coincides with the gating ED frame 0 for 97% of subjects — confirming target_t=0=ED is a genuine anchor.

Eight subjects spanning the ES-frame range. All share ED at t0 (the anchor), but the contraction trough (ES) sits at different phase indices.

End-systole is not fixed

ES frame (LV-volume minimum) across 301 subjects. Mean 5.9, std 0.72, range 3–8. Robust sub-frame ES (parabolic trough fit, immune to integer-grid quantization) std = 0.68 frames; trough near-tie fraction 85% (the LV curve is physiologically flat near ES, so the integer ES has ±1-frame uncertainty — the sub-frame estimate and the population spread below both exceed that, so the drift is real, not quantization noise).

State spread at each target_t — the direct answer

Left: LV/EDV for every subject. Right: cross-subject standard deviation of cardiac state at each target_t. If target_t were a fixed state, this would be ≈0 everywhere. Instead it is ≈0 only at the ED anchor (t0) and rises to a clear maximum in mid-systole and mid-diastole. The contraction-fraction std (timing only, EF removed) is pinned to 0 at each subject's own ED/ES endpoints by construction, so read it at the interior frames; the v_rel std (which keeps EF variation) is the more conservative drift statistic.

Caveats (from adversarial review)

  • Flat ES trough. LV velocity ≈ 0 near ES, so the volume trough is physiologically flat (~2–3 frames). The integer argmin-ES therefore has ±1-frame noise; we report a robust parabolic sub-frame ES and lead the drift claim with the continuous per-phase state-spread curve, both of which exceed the trough width.
  • Endpoint-pinned contraction fraction. cf≡1 at ED and ≡0 at ES per subject, so its std is artificially 0 at the endpoints; valid only at interior frames. v_rel (un-normalized) is reported alongside as the confound-free statistic.
  • Domain / thick-slice bias. Task114 was trained on M&Ms SAX cine; our inputs are CMRxRecon recon at 8 mm slices. A segmentation bias that is constant across a subject's 12 phases cancels in ES-frame, EF, and cf (all relative), protecting the qualitative conclusion; only a phase-dependent ES-specific bias could inflate the drift magnitude. EF≈60% indicates segmentation is in the right regime.

Part 3 — Does mixing ACDC widen the ambiguity? (the actual decision)

ACDC is the candidate dataset to mix in. It has the same gated-cine structure but is a disease cohort (DCM/HCM/MINF/NOR/RV, 150 subjects) with labeled ED/ES. We run ACDC's cine through the identical pipeline (resample each subject to 12 ED-aligned phases → same Task114 segmentation → argmin-LV ES), so the comparison is method-matched, and also read ES straight from ACDC's ground-truth labels as a cross-check.

The ground-truth ACDC distribution (orange dashed) reproduces the literature value (mean 0.38 ± 0.08, range 0.25–0.69). Method-matched, ACDC ES is both earlier and more spread than CMRxRecon (CMRxRecon nnU-Net mean 0.49, std 0.06; ACDC nnU-Net std 0.10). The two distributions peak at different target_t, so a fixed target_t means a different cardiac state depending on which dataset a sample came from — the core mixing hazard.
Why ACDC is more spread — and this uses ACDC's ground-truth ED/ES labels, not the segmentation. Pathology shifts systolic timing significantly (Kruskal–Wallis p≈3×10⁻⁶, large effect): DCM (dilated, failing — EF≈20%) reaches end-systole latest (mean 0.43, out to 0.69); HCM/NOR earliest and tightest (0.34). CMRxRecon's narrower, healthier-looking distribution (EF 64±6% vs ACDC 48±21%) has no equivalent late-ES tail.
Validation that the label-free “min-LV = ES” definition (the only one possible for CMRxRecon) is trustworthy: on ACDC, segmentation-derived ES agrees with the ground-truth ES per subject (r=0.72, 87% within ±1 phase, small +0.31-phase late bias). The outliers are diseased hearts (DCM/MINF) where the cavity-volume minimum legitimately diverges from clinical ES — so the definition is most reliable on healthy hearts, i.e. exactly the regime CMRxRecon lives in.
Per-target_t cross-subject state spread, both cohorts on the same 12-phase axis. ACDC sits above CMRxRecon across systole and mid-cycle (t1–t8) and peaks higher (~0.19–0.25 vs ~0.13–0.20) — a fixed target_t is a noisier state signal in ACDC; the curves only converge in late diastole, where CMRxRecon's later ES shifts its own spread. Mixing pools these, so the same target_t carries CMRxRecon-state and ACDC-state samples together: the bounded within-cohort ambiguity becomes a genuinely contradictory cross-cohort one.
Mixing verdict. ACDC's ES distribution is both shifted (peaks at a different target_t) and 1.3×–1.7× wider than CMRxRecon's (the range spans the conservative ground-truth-label comparison to the same-segmentation one), driven by DCM/HCM timing and a 4× wider EF spread, and it re-anchors k/N at a different native frame count (ACDC NbFrame 12–35). So mixing without physiological ED↔ES normalization (which neither dataset's labels fully support — CMRxRecon has no ES label at all) does inject contradictory target_t→state supervision — empirically confirmed, not just argued.

Methods & reproducibility

tools/cmrxrecon_phase_analysis/: structural_facts.py (exact frame counts + native TemporalPhase + resample proxy), prep_phases.py (native SAX phases → nnU-Net inputs, true per-subject spacing), analyze_phases.py (LV volume per phase → ED/ES/EF + per-phase state spread). Segmentation: nnUNet_predict -t 114 -m 2d -tr nnUNetTrainerV2_MMS in the isolated nnunet env. The analysis code was reviewed by 4 independent agents; one confirmed bug (a constant-LV subject NaN-poisoning the aggregate) was fixed, and robustness metrics (sub-frame ES, trough sharpness, dual EF) were added per review.
